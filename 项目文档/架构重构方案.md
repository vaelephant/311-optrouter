# 架构重构方案 - 按开发规范调整

## 📋 当前问题分析

### ❌ 违反规范的地方

1. **没有使用 Route Group**
   - 所有页面都在 `app/` 根目录下
   - 没有按模块组织：`app/(module-name)/`

2. **没有 Domain 层**
   - 业务逻辑直接写在 API Route 中（如 `app/api/auth/login/route.ts`）
   - 直接在 Route 中使用 `prisma`，没有 Service/Repo 分离

3. **组件组织不规范**
   - `components/admin/` 应该移到 `app/(admin)/components/`
   - 模块私有组件应该放在模块内部

4. **模块间通信不规范**
   - 没有通过 `lib/modules/` 暴露模块 API
   - 可能存在跨模块直接依赖

5. **Server Actions 缺失**
   - 只有 API Routes，没有 Server Actions
   - 不符合规范中的 "Server Actions 优先" 原则

## 📦 模块划分方案

### 1. **认证模块 `(auth)`**

**职责：**
- 用户登录
- 用户注册
- 密码管理
- 登录日志记录

**Domain 层：**
- `auth.service.ts` - 登录/注册业务流程
- `auth.repo.ts` - 用户数据访问、登录日志
- `auth.schema.ts` - 登录/注册表单校验
- `auth.types.ts` - 认证相关类型

**迁移内容：**
- `app/login/page.tsx` → `app/(auth)/login/page.tsx`
- `app/register/page.tsx` → `app/(auth)/register/page.tsx`
- `app/api/auth/login/route.ts` → 改为 Server Action 或保留 API Route
- `app/api/auth/register/route.ts` → 改为 Server Action 或保留 API Route
- `lib/auth-utils.ts` → 部分移到 `domain/auth.service.ts`，工具函数保留在 `lib/auth/utils.ts`

## 🔄 重构步骤（分阶段执行）

### 阶段 1：创建 Domain 层基础结构

1. 为每个模块创建 `domain/` 目录
2. 创建 `*.types.ts` 定义类型
3. 创建 `*.repo.ts` 封装数据访问
4. 创建 `*.service.ts` 实现业务逻辑

### 阶段 2：迁移业务逻辑

1. 将 API Route 中的业务逻辑提取到 Domain Service
2. API Route 只负责参数接收和调用 Service
3. 创建 Server Actions（如需要）

### 阶段 3：重组目录结构

1. 创建 Route Group：`(auth)`, `(dashboard)`, `(admin)`, `(invite)`, `(api-keys)`, `(usage)`
2. 移动页面到对应模块
3. 移动组件到对应模块
4. 创建 `lib/modules/` 暴露模块 API

### 阶段 4：清理和优化

1. 删除旧文件
2. 更新导入路径
3. 添加模块级错误边界
4. 完善类型定义

```typescript
// app/(auth)/domain/auth.types.ts
export interface LoginParams {
  email: string;
  password: string;
}

export interface LoginResult {
  success: true;
  userId: string;
  email: string;
  token: string;
}

// app/(auth)/domain/auth.repo.ts
export async function findUserByEmail(email: string) {
  return prisma.user.findUnique({ where: { email } });
}

export async function createLoginLog(data: LoginLogData) {
  return prisma.userLoginLog.create({ data });
}

// app/(auth)/domain/auth.service.ts
export async function loginUser(params: LoginParams): Promise<LoginResult> {
  // 1. 验证邮箱格式
  // 2. 查找用户
  // 3. 验证密码
  // 4. 记录登录日志
  // 5. 生成 Token
  // 返回结果
}

// app/(auth)/api/login/route.ts（或改为 Server Action）
export async function POST(request: NextRequest) {
  const body = await request.json();
  const result = await loginUser(body);
  return NextResponse.json(result);
}

// lib/modules/auth.ts
export * from '@/app/(auth)/domain/auth.service';
```

## ⚠️ 注意事项

1. **渐进式重构**：不要一次性重构所有模块，按模块逐步进行
2. **保持功能可用**：重构过程中确保功能正常
3. **测试覆盖**：重构后添加单元测试
4. **文档更新**：更新相关文档和注释
