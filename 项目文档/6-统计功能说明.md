## 📊 用户统计功能

### 功能概述

**路径：** `/admin/stats`  
**文件：** `app/(admin)/stats/page.tsx`

**显示内容：**
- 今日登录用户数（去重）
- 近7日登录用户数（去重）
- 今日新注册用户数
- 近7日新注册用户数
- 总登录次数（历史累计）
- 登录和注册趋势曲线图（7日）

### API 接口

**端点：** `GET /api/stats/users?days=7`  
**文件：** `app/(admin)/api/stats/users/route.ts`

**请求参数：**
- `days` - 统计天数（默认 7 天）

**请求头：**
- `x-user-id` - 当前用户 ID（用于认证）

**响应格式：**
```json
{
  "success": true,
  "data": {
    "today": {
      "login_users": 10,
      "new_users": 5,
      "visits": 50,
      "visit_users": 15
    },
    "seven_days": {
      "login_users": 50,
      "new_users": 20
    },
    "total": {
      "login_count": 1000
    },
    "daily_data": [
      {
        "date": "2024-01-01",
        "login_users": 5,
        "new_users": 2
      },
      // ... 更多日期数据
    ]
  }
}
```

### 业务逻辑

**Service：** `admin.service.ts` → `fetchUserStats`

**统计逻辑：**

1. **今日统计**
   - 今日新注册用户：查询 `users` 表，`createdAt` 在今日 0 点至明日 0 点之间
   - 今日登录用户：查询 `user_login_logs` 表，`loginAt` 在今日 0 点至明日 0 点之间，按 `userId` 去重

2. **近7日统计**
   - 近7日新注册用户：查询 `users` 表，`createdAt` 在 7 天前至今日之间
   - 近7日登录用户：查询 `user_login_logs` 表，`loginAt` 在 7 天前至今日之间，按 `userId` 去重

3. **总登录次数**
   - 查询 `user_login_logs` 表的总记录数

4. **每日数据**
   - 循环计算每一天的新注册用户数和登录用户数（去重）
   - 使用 UTC 时间计算，避免时区偏移问题

### 前端组件

**UserStatsCards：** `app/(admin)/components/user-stats-cards.tsx`
- 显示 5 个统计卡片
- 高亮显示登录相关统计（红色边框）

**UserStatsTrendChart：** `app/(admin)/components/user-stats-trend-chart.tsx`
- 使用 `recharts` 的 `AreaChart` 显示趋势
- 两条曲线：登录用户数（红色）、新注册用户数（蓝色）
- 支持实时刷新

## 📈 使用统计功能

### 功能概述

**路径：** `/dashboard`  
**文件：** `app/(dashboard)/dashboard/page.tsx`

**显示内容：**
- API 请求总量
- 消费金额
- 平均延迟
- 成功率
- 使用趋势图表（按日期）
- 模型使用分布（饼图）

### API 接口

**端点：** `GET /api/usage?days=30&model=xxx`  
**文件：** `app/(dashboard)/api/usage/route.ts`

**请求参数：**
- `days` - 统计天数（1-365，默认 30）
- `model` - 模型名称（可选，用于筛选）

**请求头：**
- `x-user-id` - 当前用户 ID（用于认证）

**响应格式：**
```json
{
  "period": {
    "days": 30,
    "start": "2024-01-01T00:00:00Z",
    "end": "2024-01-31T23:59:59Z"
  },
  "summary": {
    "total_requests": 1000,
    "total_tokens": 500000,
    "total_cost": 10.50,
    "avg_latency_ms": 250,
    "success_rate": 0.98
  },
  "daily": [
    {
      "date": "2024-01-01",
      "tokens": 10000,
      "requests": 50,
      "cost": 0.50
    },
    // ... 更多日期数据
  ],
  "models": {
    "gpt-4": {
      "tokens": 200000,
      "cost": 5.00,
      "requests": 200
    },
    // ... 更多模型数据
  }
}
```

### 业务逻辑

**Service：** `dashboard.service.ts` → `fetchUsageStats`

**统计逻辑：**

1. **查询使用日志**
   - 从 `usage_logs` 表查询指定时间范围内的记录
   - 按 `userId` 筛选（只查询当前用户的数据）
   - 可选按 `model` 筛选

2. **汇总统计**
   - 总请求数：记录总数
   - 总 Token 数：`inputTokens + outputTokens` 累加
   - 总成本：`cost` 累加
   - 平均延迟：`latencyMs` 的平均值
   - 成功率：`status === 'success'` 的记录数 / 总记录数

3. **按日期分组**
   - 将记录按 `createdAt` 的日期分组
   - 计算每天的 tokens、requests、cost

4. **按模型分组**
   - 将记录按 `model` 分组
   - 计算每个模型的 tokens、cost、requests

### 前端组件

**StatCards：** `app/(dashboard)/components/stat-cards.tsx`
- 显示 4 个统计卡片：请求总量、消费金额、平均延迟、成功率
- 实时数据，从 API 获取

**UsageChart：** `app/(dashboard)/components/usage-chart.tsx`
- 使用 `recharts` 的 `LineChart` 显示使用趋势
- 显示 tokens、requests、cost 三条曲线

**ModelUsage：** `app/(dashboard)/components/model-usage.tsx`
- 使用 `recharts` 的 `PieChart` 显示模型使用分布
- 显示每个模型的 token 使用量和占比

## 📱 设备统计功能

### API 接口

**端点：** `GET /api/admin/devices?days=7`  
**文件：** `app/(admin)/api/admin/devices/route.ts`

**功能：**
- 统计不同设备类型的登录次数
- 设备类型：Desktop、Mobile、Tablet、Unknown

**业务逻辑：**
- 从 `user_login_logs` 表查询指定时间范围内的记录
- 解析 `userAgent` 字段，判断设备类型
- 按设备类型分组统计

**设备类型判断逻辑：**
```typescript
// 桌面设备：包含 windows/macintosh/linux/x11，且不包含 mobile/android
// 平板设备：包含 ipad/tablet/playbook
// 移动设备：包含 mobile/android/iphone/ipod/blackberry/opera mini/windows phone
// 未知设备：其他情况
```

## 🌍 地点统计功能

### API 接口

**端点：** `GET /api/admin/locations?days=7`  
**文件：** `app/(admin)/api/admin/locations/route.ts`

**功能：**
- 统计不同地理位置的登录次数
- 从 IP 地址解析地理位置（当前为简化实现）

**业务逻辑：**
- 从 `user_login_logs` 表查询指定时间范围内的记录
- 解析 `ipAddress` 字段，判断地理位置
- 按地理位置分组统计

**注意：** 当前实现为简化版本，实际生产环境应使用 IP 地理位置数据库（如 MaxMind GeoIP2）。

## ⏱️ 登录时长统计功能

### API 接口

**端点：** `GET /api/admin/login-duration?days=7`  
**文件：** `app/(admin)/api/admin/login-duration/route.ts`

**功能：**
- 统计用户登录后的会话时长
- 计算平均会话时长、最长会话时长、最短会话时长

**业务逻辑：**
- 从 `user_behavior_logs` 表查询指定时间范围内的记录
- 计算 `durationSeconds`（`endTime - startTime`）
- 统计平均、最大、最小会话时长

## 🎯 用户行为统计功能

### API 接口

**端点：** `GET /api/admin/behavior?days=7`  
**文件：** `app/(admin)/api/admin/behavior/route.ts`

**功能：**
- 统计用户功能使用情况
- 统计不同功能的访问次数和使用时长

**业务逻辑：**
- 从 `user_behavior_logs` 表查询指定时间范围内的记录
- 按 `functionName` 分组统计
- 计算每个功能的访问次数、总使用时长、平均使用时长

### 行为追踪 API

**端点：** `POST /api/user-behavior/track`  
**文件：** `app/(dashboard)/api/user-behavior/track/route.ts`

**请求体：**
```json
{
  "user_id": "uuid-string",
  "email": "user@example.com",
  "function_name": "dashboard",
  "action": "enter"  // 或 "leave"
}
```

**功能：**
- 记录用户进入功能（`action: "enter"`）
- 记录用户离开功能（`action: "leave"`）
- 自动计算会话时长（离开时计算 `endTime - startTime`）

**业务逻辑：**
- `enter`：创建新的行为日志记录，设置 `startTime`
- `leave`：查找对应的行为日志记录，设置 `endTime` 和 `durationSeconds`

## 📝 数据表说明

### user_login_logs 表

**用途：** 记录用户登录信息

**字段：**
- `id` - BigInt（主键）
- `user_id` - UUID（外键，关联 users）
- `email` - 用户邮箱
- `login_at` - 登录时间
- `ip_address` - IP 地址
- `user_agent` - User-Agent

**索引：**
- `user_id`
- `email`
- `login_at`

### user_behavior_logs 表

**用途：** 记录用户功能使用行为

**字段：**
- `id` - BigInt（主键）
- `user_id` - UUID（外键，关联 users）
- `email` - 用户邮箱
- `function_name` - 功能名称
- `start_time` - 开始时间
- `end_time` - 结束时间（可选）
- `duration_seconds` - 使用时长（秒）
- `created_at` - 创建时间

**索引：**
- `user_id`
- `email`
- `function_name`
- `start_time`

### usage_logs 表

**用途：** 记录 API 使用情况

**字段：**
- `id` - BigInt（主键）
- `user_id` - UUID（外键，关联 users）
- `api_key_id` - UUID（外键，关联 api_keys，可选）
- `model` - 模型名称
- `input_tokens` - 输入 Token 数
- `output_tokens` - 输出 Token 数
- `total_tokens` - 总 Token 数
- `cost` - 本次调用费用
- `latency_ms` - 延迟（毫秒）
- `status` - 状态（success/error/rate_limited）
- `created_at` - 创建时间

**索引：**
- `[user_id, created_at]`
- `model`

## 📝 注意事项

1. **时区处理**
   - 所有日期计算使用 UTC 时间，避免时区偏移
   - 前端显示时转换为本地时区

2. **性能优化**
   - 统计查询使用数据库索引
   - 大量数据查询时考虑分页或限制时间范围

3. **数据准确性**
   - 登录用户数使用去重统计（DISTINCT user_id）
   - 总登录次数统计所有登录记录（不去重）

4. **设备类型识别**
   - 基于 User-Agent 字符串解析，可能不够准确
   - 生产环境可考虑使用专业库（如 `ua-parser-js`）

5. **地理位置识别**
   - 当前为简化实现，生产环境应使用 IP 地理位置数据库
