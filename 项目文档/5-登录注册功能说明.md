# 5-登录注册功能说明

## 📋 概述

本文档详细说明用户登录和注册功能的实现方式、API 接口、业务流程和技术细节。

**模块位置：** `app/(auth)/`

## 🔐 登录功能

### 前端页面

**路径：** `/login`  
**文件：** `app/(auth)/login/page.tsx`

**功能：**
- 邮箱输入验证
- 密码输入
- 登录表单提交
- 登录成功后保存用户信息到 `localStorage`
- 自动重定向到 `/dashboard`

**关键代码：**
```typescript
// 登录成功后保存用户信息
saveUserAuth(userId, data.email, data.token)
router.push('/dashboard')
```

### API 接口

**端点：** `POST /api/auth/login`  
**文件：** `app/(auth)/api/auth/login/route.ts`

**请求体：**
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

**成功响应（200）：**
```json
{
  "success": true,
  "message": "登录成功",
  "user_id": "uuid-string",
  "email": "user@example.com",
  "token": "jwt-token-string"
}
```

**失败响应（401）：**
```json
{
  "success": false,
  "detail": "邮箱或密码错误"
}
```

**内部错误（500）：**
```json
{
  "success": false,
  "detail": "数据库连接失败，请稍后重试"
}
```

### 业务流程

1. **前端提交登录表单**
   - 验证邮箱格式
   - 发送 POST 请求到 `/api/auth/login`

2. **后端处理（auth.service.ts）**
   - 验证邮箱格式
   - 查找用户（`findUserByEmail`）
   - 验证密码（`verifyPassword` - bcryptjs）
   - 更新最后登录时间
   - 记录登录日志（IP、User-Agent、时间）
   - 生成 Token（`createSimpleToken`）

3. **返回结果**
   - 成功：返回 `user_id`、`email`、`token`
   - 失败：返回错误信息

4. **前端处理**
   - 成功：保存到 `localStorage`，重定向到 `/dashboard`
   - 失败：显示错误信息

### 登录日志记录

**表：** `user_login_logs`

**记录内容：**
- `user_id` - 用户 ID

- `ip_address` - IP 地址（从请求头提取）
- `user_agent` - User-Agent（从请求头提取）

## 📝 注册功能

### 前端页面

**路径：** `/register`  
**文件：** `app/(auth)/register/page.tsx`

**功能：**
- 邮箱输入验证
- 密码输入和确认
- 邀请码输入（可选，支持 URL 参数）
- 注册表单提交
- 注册成功后保存用户信息到 `localStorage`
- 自动重定向到 `/dashboard`

**邀请码支持：**
- URL 参数：`/register?invite_code=XXXX`
- 表单输入：用户手动输入邀请码

**关键代码：**
```typescript
// 从 URL 参数或表单中获取邀请码
const urlInviteCode = searchParams.get('invite_code')
const finalInviteCode = inviteCode?.trim() || urlInviteCode?.trim().toUpperCase() || undefined
```

### API 接口

**端点：** `POST /api/auth/register`  
**文件：** `app/(auth)/api/auth/register/route.ts`

**请求体：**
```json
{
  "email": "user@example.com",
  "password": "password123",
  "invite_code": "XXXX"  // 可选
}
```

**成功响应（200）：**
```json
{
  "success": true,
  "message": "注册成功",
  "user_id": "uuid-string",
  "email": "user@example.com",
  "token": "jwt-token-string"
}
```

**失败响应（400）：**
```json
{
  "success": false,
  "detail": "该邮箱已注册"
}
```

或

```json
{
  "success": false,
  "detail": "邮箱格式不正确"
}
```

### 业务流程

1. **前端提交注册表单**
   - 验证邮箱格式
   - 验证密码长度（至少 6 位）
   - 验证两次密码是否一致
   - 发送 POST 请求到 `/api/auth/register`

2. **后端处理（auth.service.ts）**
   - 验证邮箱格式
   - 验证密码长度
   - 检查邮箱是否已存在（`checkEmailExists`）
   - 创建新用户（`createUser`）
     - 密码哈希（`hashPassword` - bcryptjs）
     - 生成 UUID 作为用户 ID
   - 处理邀请码（如果提供）
     - 调用 `handleInviteCodeOnRegister`（来自 invite 模块）
     - 创建邀请关系
     - 更新邀请码使用次数
   - 生成 Token

3. **返回结果**
   - 成功：返回 `user_id`、`email`、`token`
   - 失败：返回错误信息

4. **前端处理**
   - 成功：保存到 `localStorage`，重定向到 `/dashboard`
   - 失败：显示错误信息

### 邀请码集成

**集成点：** `app/(auth)/api/auth/register/route.ts`

**处理流程：**
```typescript
// 注册成功后，如果提供了邀请码，调用 invite 模块的处理函数
if (params.inviteCode && onInviteCodeProcess) {
  await onInviteCodeProcess(user.id, trimmedEmail, params.inviteCode)
}
```

**邀请码处理（invite.service.ts）：**
- 验证邀请码有效性
- 创建邀请关系（`InviteRelation`）
- 更新邀请码使用次数
- 检查并发放奖励（如果达到奖励条件）

## 🔒 安全特性

### 密码安全

- **哈希算法：** bcryptjs
- **密码存储：** 数据库中只存储哈希值，不存储明文
- **密码验证：** 使用 `bcrypt.compare()` 进行验证

**函数：** `createSimpleToken`（`lib/auth-utils.ts`）

**实现：**
```typescript

```

### 路由守卫

**组件：** `AuthGuard`（`app/(auth)/components/auth-guard.tsx`）

**功能：**
- 检查用户是否已登录（通过 `localStorage`）
- 未登录用户自动重定向到 `/login`
- 防止 SSR 时的 hydration 错误

**使用方式：**
```typescript

```

## 💾 数据存储

### 用户信息（localStorage）

**键名：**
- `user_id` - 用户 ID（UUID）
- `email` - 用户邮箱
- `token` - 认证 Token

**工具函数（lib/auth-client.ts）：**
- `saveUserAuth(userId, email, token)` - 保存用户信息
- `getCurrentUserId()` - 获取当前用户 ID
- `getCurrentUserEmail()` - 获取当前用户邮箱
- `isAuthenticated()` - 检查是否已登录
- `clearUserAuth()` - 清除用户信息（登出）

### 数据库表

**users 表：**
- `id` - UUID（主键）
- `email` - 邮箱（唯一索引）
- `password` - 密码哈希
- `role` - 用户角色（默认 "user"）
- `createdAt` - 创建时间
- `updatedAt` - 更新时间

**user_login_logs 表：**
- `id` - BigInt（主键）
- `user_id` - 用户 ID（外键）
- `email` - 用户邮箱
- `login_at` - 登录时间
- `ip_address` - IP 地址
- `user_agent` - User-Agent

## 🔄 错误处理

### 前端错误处理

**登录页面：**
- 网络错误：显示 "网络错误，请稍后重试"
- 认证失败：显示后端返回的 `detail` 信息
- 缺少字段：显示 "登录返回缺少 user_id，请检查后端返回字段"

**注册页面：**
- 密码不一致：显示 "两次输入的密码不一致"
- 密码长度不足：显示 "密码长度至少为6位"
- 其他错误：显示后端返回的 `detail` 信息

### 后端错误处理

**状态码：**
- `200` - 成功
- `400` - 参数错误（邮箱格式、密码长度、已注册）
- `401` - 认证失败（邮箱或密码错误）
- `500` - 内部错误（数据库连接失败等）

**错误信息：**
- 邮箱格式不正确
- 密码长度至少为6位
- 该邮箱已注册
- 邮箱或密码错误
- 登录失败，请稍后重试

## 🛠️ 技术栈

- **前端框架：** Next.js 16 (App Router)
- **UI 组件：** shadcn/ui
- **状态管理：** React Hooks (useState, useEffect)
- **路由：** Next.js App Router
- **密码哈希：** bcryptjs
- **数据库：** PostgreSQL (Prisma ORM)
- **认证：** 简单 Token（Base64 编码）

## 📝 注意事项

1. **Token 安全性**
   - 当前使用简单的 Base64 编码，生产环境应使用 JWT
   - Token 存储在 `localStorage`，存在 XSS 风险

2. **密码策略**
   - 当前最小长度为 6 位，建议增强（大小写、数字、特殊字符）

3. **邀请码处理**
   - 邀请码处理失败不影响注册流程（只记录错误日志）
   - 邀请码验证在注册成功后进行

4. **登录日志**
   - 每次登录都会记录 IP 和 User-Agent
   - 可用于安全审计和异常检测
