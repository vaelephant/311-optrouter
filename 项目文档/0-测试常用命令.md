# 测试常用命令

## 测试 OPENAI_API_KEY 是否可用（直连 OpenAI，用 curl）

先加载 key（在项目根目录执行）：

```bash
export OPENAI_API_KEY="你的key"
```

然后执行（列出模型 = 校验 key）：

```bash
curl -s -H "Authorization: Bearer $OPENAI_API_KEY" https://api.openai.com/v1/models
```

返回一长串 JSON 即表示 key 有效；若 key 无效会返回 401 等错误 JSON。

---

## 非流式（一次返回完整 JSON，可用 jq 格式化）

请求体里 `"stream": false`（或省略），返回整段 JSON：

```bash
curl -X POST http://localhost:3001/v1/chat/completions \
  -H "Authorization: Bearer sk-mm61wdhk-2fcvs5uq9he" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"请输出 4 个字"}],"stream":false}' | jq
```

## 流式（只输出生成文字，请求格式同上，仅改 stream: true）

请求体里 `"stream": true`，其它格式不变；返回为**纯文本流**（只含模型生成的文字，无 JSON）：

```bash
curl -N -X POST http://localhost:3001/v1/chat/completions \
  -H "Authorization: Bearer sk-mm61wdhk-2fcvs5uq9he" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"请输出 4 个字"}],"stream":true}'
```

- `-N`：边收边打印。流末尾会多一行：**生成时间(ms)、prompt_tokens、completion_tokens、费用(元)**。

# 计费测试

## 测 Ollama（本地模型）

需本地已启动 Ollama 且 `config/models.toml` 里已配置对应模型（如 `qwen2.5:7b-instruct`）。Bearer 替换为你的 API Key。



```bash

curl -X POST http://localhost:3001/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-mm61wdhk-2fcvs5uq9he" \
  -d '{
    "model": "qwen2.5:7b-instruct",
    "messages": [{"role": "user", "content": "请输出 10个字"}],
    "stream": true
  }'

```
