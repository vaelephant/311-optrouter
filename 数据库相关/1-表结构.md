🗄 一、核心数据表总览

users                ← 用户

⸻

👤 二、users（用户表）

CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email TEXT UNIQUE NOT NULL,
    password TEXT,
    role TEXT DEFAULT 'user',
    created_at TIMESTAMP DEFAULT NOW()
);


⸻

🔑 三、api_keys（API密钥）

CREATE TABLE api_keys (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    key_hash TEXT NOT NULL,
    name TEXT,
    status TEXT DEFAULT 'active',
    rate_limit_per_min INT DEFAULT 60,
    created_at TIMESTAMP DEFAULT NOW()
);

👉 存 hash，不存明文 key。

⸻

🤖 四、model_pricing（模型价格）

👉 可随时调整价格 & 利润率

CREATE TABLE model_pricing (
    model_name TEXT PRIMARY KEY,
    input_price NUMERIC(10,6),   -- 每1K tokens价格
    output_price NUMERIC(10,6),
    created_at TIMESTAMP DEFAULT NOW()
);

示例数据：

INSERT INTO model_pricing VALUES
('gpt-4o', 0.005, 0.015, NOW()),
('deepseek-chat', 0.0005, 0.001, NOW());


⸻

📊 五、usage_logs（计费核心表）

👉 这是整个平台最重要的数据表

CREATE TABLE usage_logs (
    id BIGSERIAL PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    api_key_id UUID REFERENCES api_keys(id),
    model TEXT,
    input_tokens INT,
    output_tokens INT,
    total_tokens INT,
    cost NUMERIC(10,6),
    latency_ms INT,
    status TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);


⸻

为什么记录 latency & status？

✔ SLA 监控
✔ 失败率分析
✔ 企业客户报告

👉 企业级功能必备。

⸻

💰 六、user_balances（用户余额）

👉 预付费模式核心

CREATE TABLE user_balances (
    user_id UUID PRIMARY KEY REFERENCES users(id),
    balance NUMERIC(12,4) DEFAULT 0,
    updated_at TIMESTAMP DEFAULT NOW()
);


⸻

💳 七、transactions（财务流水）

👉 所有资金变化必须记录

CREATE TABLE transactions (
    id BIGSERIAL PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    amount NUMERIC(12,4),
    type TEXT,  -- recharge / usage / refund
    description TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);


⸻

示例：

type	amount	说明
recharge	+20	用户充值
usage	-0.02	AI调用扣费

👉 财务对账必须依赖此表。

⸻

📦 八、plans（套餐定义）

CREATE TABLE plans (
    id SERIAL PRIMARY KEY,
    name TEXT,
    monthly_price NUMERIC(10,2),
    token_quota BIGINT,
    created_at TIMESTAMP DEFAULT NOW()
);


⸻

🧾 九、subscriptions（用户订阅）

CREATE TABLE subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    plan_id INT REFERENCES plans(id),
    current_period_end TIMESTAMP,
    status TEXT DEFAULT 'active',
    created_at TIMESTAMP DEFAULT NOW()
);

⚡ 十一、关键索引优化（必须）

提高统计速度 👇

CREATE INDEX idx_usage_user_time
ON usage_logs(user_id, created_at);

CREATE INDEX idx_usage_model
ON usage_logs(model);

CREATE INDEX idx_transactions_user
ON transactions(user_id);

👉 否则后期统计会变慢。

⸻

📊 十二、常用统计查询

用户今日消费：

SELECT SUM(cost)
FROM usage_logs
WHERE user_id = $1
AND created_at > CURRENT_DATE;


⸻

每个模型使用量：

SELECT model, SUM(total_tokens)
FROM usage_logs
GROUP BY model;


⸻

平台利润分析：

SELECT SUM(cost) FROM usage_logs;

🔐 十三、安全建议（生产必须）

✅ API key 存 hash
✅ 财务流水不可修改
✅ 使用事务处理扣费
✅ 余额不能为负
✅ 设置数据库约束

示例：

CHECK (balance >= 0)
